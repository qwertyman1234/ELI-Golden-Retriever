# What is Git?

## The Explanation

Git is code lore storage with an UNDO button. Every time the pack does a good change, we take a snapshot called a commit, like saving a pic of me doing a perfect sit. If we want to try a risky trick, we make a branch, which is basically a parallel universe walk route. If the new route is fire, we merge it back into the main walk. If it is a mud speedrun, we just roll back to an earlier snapshot and pretend the chaos never happened!!

## Key Terms

- Commit: A saved snapshot, aka the "we did a thing" receipt
- Branch: A side quest timeline for experiments
- Merge: Combine the side quest back into main story
- Repository: The whole yard where the snapshots live
- Remote: The neighbor yard we sync with

## Why This Is Actually A Big Deal

Git lets a whole pack work on the same project without yeeting each other's changes. You get history, collaboration, and time travel to undo mistakes. For humans this is productivity. For me it is MAXIMUM good boy energy.
